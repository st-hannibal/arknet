Developer Guide

Build applications on arknet using the OpenAI-compatible API. Any existing OpenAI integration works by changing base_url.

API Reference

arknet exposes the standard OpenAI endpoints:

EndpointMethodDescription
/v1/chat/completionsPOSTChat completion (streaming + non-streaming)
/v1/modelsGETList available models
/healthGETNode health status
/v1/statusGETChain status (height, peers, consensus)
/v1/txPOSTSubmit a signed transaction

Chat completions

Non-streaming

curl -X POST http://localhost:26657/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3.1-8B-Instruct",
    "messages": [{"role": "user", "content": "Explain arknet in one sentence"}],
    "max_tokens": 100
  }'

Streaming

curl -X POST http://localhost:26657/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3.1-8B-Instruct",
    "messages": [{"role": "user", "content": "Hello"}],
    "stream": true
  }'

Streaming returns SSE events with data: prefixed JSON chunks, terminated by data: [DONE].

SDKs

Install

# Python
pip install arknet-sdk

# TypeScript / Node.js
npm install arknet-sdk

# Rust
cargo add arknet-sdk

Python

from arknet_sdk import Client

# Auto-discover a gateway (recommended for network usage)
client = Client.connect()

# Or connect to a specific node
client = Client("http://localhost:26657")
resp = client.chat_completion(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(resp["choices"][0]["message"]["content"])

# Streaming
for chunk in client.chat_completion_stream(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Tell me a story"}],
):
    delta = chunk["choices"][0]["delta"]
    if "content" in delta:
        print(delta["content"], end="")

TypeScript

import { ArknetClient } from "arknet-sdk";

const client = new ArknetClient("http://localhost:26657");

// Non-streaming
const resp = await client.chatCompletion({
  model: "meta-llama/Llama-3.1-8B-Instruct",
  messages: [{ role: "user", content: "Hello!" }],
});
console.log(resp.choices[0].message.content);

// Streaming
for await (const chunk of client.chatCompletionStream({
  model: "meta-llama/Llama-3.1-8B-Instruct",
  messages: [{ role: "user", content: "Tell me a story" }],
})) {
  process.stdout.write(chunk.choices[0].delta.content ?? "");
}

Rust

let client = arknet_sdk::Client::new("http://localhost:26657")?;
let resp = client.chat_completion(arknet_sdk::ChatRequest {
    model: "meta-llama/Llama-3.1-8B-Instruct".into(),
    messages: vec![arknet_sdk::Message {
        role: "user".into(),
        content: "Hello!".into(),
    }],
    ..Default::default()
}).await?;
println!("{}", resp.choices[0].message.content);

Raw OpenAI SDK (advanced)

If you already have an OpenAI integration, point it at any arknet node. You'll need to know a node URL — the arknet SDKs above handle this automatically via Client.connect().

from openai import OpenAI

# Local node (free, no wallet needed)
client = OpenAI(base_url="http://localhost:26657/v1", api_key="local")

# Remote node (find URLs via /v1/gateways on any running node)
# client = OpenAI(base_url="https://some-gateway:26657/v1", api_key="ark1...")
response = client.chat.completions.create(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Hello from arknet"}],
)

On-chain transactions

Submit borsh-encoded signed transactions via /v1/tx:

curl -X POST http://localhost:26657/v1/tx \

Building a dApp

A typical arknet dApp flow:

  1. List models — GET /v1/models to show available models to your users
  2. Submit inference — POST /v1/chat/completions with the user's prompt
  3. Stream response — Parse SSE chunks and render tokens as they arrive
  4. Check status — GET /v1/status to show chain health in your UI

The API is identical to OpenAI, so any existing OpenAI-based application works with zero code changes beyond pointing base_url at an arknet node.

Privacy

  • Transport — all P2P traffic uses libp2p Noise encryption
  • Prompt isolation — prompts travel only to the assigned compute node, never through consensus or gossip
  • RPC binding — default 127.0.0.1 only; operators control exposure
  • Confidential inference (TEE) — pass prefer_tee: true to route only to nodes running inside Intel TDX / AMD SEV-SNP enclaves. Prompts are encrypted to the enclave's public key (X25519 + ChaCha20-Poly1305) — even the host OS cannot read them
  • Local mode — run your own compute node for zero-trust privacy. No network, no tokens, prompts never leave your machine

See the Privacy & Security page for the full three-tier privacy model, TEE operator guide, and comparison with centralized providers.

Confidential inference example

from arknet_sdk import Client

# Auto-discover an HTTPS + TEE gateway
client = Client.connect(require_https=True)
response = client.chat_completion(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Confidential question"}],
    prefer_tee=True,      # TEE enclaves only
    require_https=True,   # HTTPS gateways only
)

When prefer_tee is set, the router only picks TEE-capable nodes. If none are available, you get a clear error — no silent downgrade.
