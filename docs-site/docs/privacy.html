Privacy & Security

arknet is designed so your prompts stay private at every layer — from the user's device to the compute node and back. Three tiers of protection, from transport encryption to hardware-enforced confidentiality.

Three-tier privacy model

Tier Protection Available at genesis What it protects
Transport encryption Noise protocol (libp2p) Yes All P2P traffic encrypted. Eavesdroppers on the network see nothing.
Prompt isolation Protocol design Yes Only the assigned compute node sees your prompt. Validators, routers, verifiers see hashes and token counts — never content. Consensus never touches prompt data.
Confidential inference (TEE) Hardware enclaves (Intel TDX / AMD SEV-SNP) Yes (protocol ready, hardware-dependent) Even the compute node's host OS cannot see your prompt. The model runs inside a Trusted Execution Environment — encrypted memory that only the CPU can access.

How prompt encryption works (TEE)

When a user sends a request to a TEE-capable compute node, the prompt never exists in plaintext outside the hardware enclave.

User                          Enclave (inside TEE)         Host OS
  |                                |                          |
  | 1. Get enclave public key      |                          |
  |   (from chain state)           |                          |
  |<----------------------------->|                          |
  |                                |                          |
  | 2. Encrypt prompt to           |                          |
  |    enclave pubkey              |                          |
  |    (X25519 + ChaCha20)         |                          |
  |------------------------------->| 3. Decrypt inside        |
  |                                |    enclave               | sees: encrypted blob
  |                                | 4. Run inference         | sees: nothing
  |                                |    (in enclave memory)   |
  | 5. Tokens stream back          |                          |
  |    over Noise-encrypted P2P    |                          |
  |<-------------------------------|                          | sees: encrypted blob

The full flow:

  1. User generates an ephemeral X25519 keypair — one per request, discarded after.
  2. ECDH with the enclave's public key produces a shared secret. The enclave's pubkey is published on-chain when the node registers.
  3. HKDF-SHA256 derives a symmetric key from the shared secret.
  4. ChaCha20-Poly1305 encrypts the prompt — authenticated encryption, tamper-evident.
  5. The enclave decrypts using its private key, which was generated inside the TEE and never leaves enclave memory.
  6. Inference runs entirely in enclave memory. The host OS, hypervisor, and other processes cannot read it.
  7. Response tokens stream back over the Noise-encrypted P2P channel.

Cryptography: All primitives are from audited Rust crates — x25519-dalek, chacha20poly1305, hkdf, sha2. No custom cryptography.

Using confidential inference

Set prefer_tee=True and the router will only dispatch your request to TEE-capable compute nodes.

arknet Python SDK (recommended)

from arknet_sdk import Client

# Auto-discovers HTTPS gateways from on-chain registry
client = Client.connect(require_https=True)
response = client.chat_completion(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Confidential question here"}],
    prefer_tee=True,
    require_https=True,
)

Raw OpenAI SDK (advanced)

If using the OpenAI SDK directly, point at any arknet node. You'll need a gateway URL — the arknet SDK above finds one automatically.

from openai import OpenAI

client = OpenAI(base_url="http://localhost:26657/v1", api_key="local")
response = client.chat.completions.create(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Confidential question here"}],
    extra_body={"prefer_tee": True, "require_https": True},
)

No silent downgrades. When prefer_tee=True, the router only picks TEE-capable nodes. If none are available, you get a clear error — the request is never silently routed to a non-TEE node.

For node operators: enabling TEE

If your hardware supports Intel TDX or AMD SEV-SNP, you can register as a TEE-capable compute node in five steps.

  1. Generate enclave keypair: 
    arknet tee keygen
  2. Get attestation quote from your TEE hardware. This proves your CPU supports a genuine enclave.
  3. Register on-chain: 
    arknet tee register --platform intel-tdx --quote-file quote.bin
  4. Enable in config (~/.arknet/node.toml): 
    [tee]
enabled = true
platform = "intel-tdx"
  5. Restart your node. It is now advertised as TEE-capable and receives confidential inference requests.

TEE economics

RuleDetail
1.5x emission rewardsTEE-verified inference jobs earn 50% more ARK than standard jobs. This incentivizes nodes to run confidential inference.
Fake attestation = 100% slashSubmitting a fraudulent TEE attestation (FakeTeeAttestation offense) results in total loss of staked ARK. No warnings, no partial slash.
No registration depositTEE registration is free. Unlike model registration (which costs 10,000 ARK as a spam deterrent), enabling TEE has no deposit requirement.

What TEE does NOT protect against

Honesty about limitations:

  • Metadata is visible. TEE does not hide which model you are querying or how many tokens were generated. These values appear in inference receipts — they are required for verification and reward calculation.
  • CPU side-channel attacks. TEE does not protect against side-channel vulnerabilities in specific CPU implementations (e.g., speculative execution attacks). Intel and AMD patch these as they are discovered, but they exist in the threat model.
  • Genesis-era attestation is structural. At genesis, TEE quote verification checks the quote structure and signature format. Full root-of-trust verification against Intel/AMD certificate authorities will be activated once the verification library is deployed and audited. This is a known gap documented in the security model.

HTTPS gateways: last-mile encryption

When you connect to a public gateway, the "last mile" between your device and the gateway can be observed by your ISP, wifi operator, or VPN provider. HTTPS protects this segment with TLS.

ConnectionLast mileP2P layerPrompt content
HTTP gatewayPlaintextNoise-encryptedVisible to network observers on last mile
HTTPS gatewayTLS-encryptedNoise-encryptedProtected everywhere
HTTPS + TEETLS-encryptedNoise-encryptedProtected + enclave-encrypted

For users

from arknet_sdk import Client

client = Client.connect(require_https=True)  # auto-discover HTTPS gateway
response = client.chat_completion(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Private question"}],
    require_https=True,  # only HTTPS gateways
)

When require_https is set, the router only picks HTTPS gateways. No silent downgrade — if no HTTPS gateway is available, you get a clear error.

For operators

HTTPS gateways earn a 1.2x reward multiplier on routed jobs. Setup:

  1. Get a domain and a Let's Encrypt TLS certificate (free)
  2. Configure a reverse proxy (nginx/caddy) to terminate TLS and forward to localhost:26657
  3. Register: arknet gateway register --url https://rpc.mynode.com --https

HTTPS + TEE multipliers stack: a job routed through an HTTPS gateway to a TEE node earns 1.8x emission (1.2 x 1.5).

Gateway auto-discovery

SDKs can discover gateways automatically from the on-chain registry:

# Python — auto-discover and connect
from arknet_sdk import Client
client = Client.connect(require_https=True)

# TypeScript — auto-discover and connect
const client = await ArknetClient.connect({ requireHttps: true });

No hardcoded URLs needed. The SDK contacts a seed node, reads /v1/gateways, and picks the best HTTPS gateway.

Full privacy: run your own node

For absolute privacy with zero trust required, run inference locally. No network interaction, no tokens spent, no prompts leaving your machine.

arknet start --role compute
  • 100% local inference — your hardware, your models, your data.
  • No network interaction — the node does not connect to peers.
  • No cost — no ARK tokens spent for local inference.
  • Free forever — this is a feature, not a limitation.

Point any OpenAI-compatible client at http://localhost:26657/v1 and use it like any other local inference server.
