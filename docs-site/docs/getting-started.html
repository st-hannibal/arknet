Getting Started

Three steps to join the arknet network. Total time: under 5 minutes.

1. Install

curl -fsSL https://arknet.arkengel.com/install.sh | sh

Or build from source:

git clone https://github.com/st-hannibal/arknet.git
cd arknet
cargo build --release
cp target/release/arknet /usr/local/bin/

2. Initialize

arknet init --network mainnet

This creates ~/.arknet/ with your node identity (keypair) and default config. Your key is generated once and persisted — it's your node's permanent identity on the network.

3. Configure and start

Edit ~/.arknet/node.toml to pick your role:

Compute node (needs GPU)

[roles]
compute = true

[resources.compute]
gpu_devices = [0]
max_vram_gb = 8
loaded_models = ["meta-llama/Llama-3.1-8B-Instruct:Q4_K_M"]
arknet start --role compute

Router (any hardware)

[roles]
router = true
arknet start --role router

Validator (needs stake)

[roles]
validator = true
arknet start --role validator

Hardware guide: A Raspberry Pi 5 can run a router. A laptop with an RTX 3060 can serve 7B models. A desktop with an RTX 4090 can serve 27B models. Match the model tier to your VRAM — see Genesis Models.

Networking

Your node needs one public port — the P2P port (default 26656). Open it in your firewall or router. Everything else stays private.

PortDefaultPublic?Purpose
P2P26656YesNode discovery, block gossip, inference dispatch. Noise-encrypted.
RPC26657No (localhost)Your personal dashboard — balance, explorer, submit transactions.
Metrics9090No (localhost)Prometheus scrape endpoint for monitoring.

This is the same as every blockchain node. Bitcoin: port 8333. Ethereum: 30303. Cosmos: 26656. Open the P2P port, the rest is private.

Want to run a public gateway? Change rpc_listen to "0.0.0.0:26657" in node.toml. Users can send inference requests through you and you earn the 5% router cut. This is how community gateways work — just a node with a public RPC.

Register your gateway on-chain so SDKs discover it automatically. HTTPS gateways earn a 1.2x reward multiplier:

# HTTP gateway (any machine)
arknet gateway register --url http://YOUR_IP:26657

# HTTPS gateway (domain + TLS cert — 1.2x rewards)
arknet gateway register --url https://rpc.mynode.com --https

See Privacy & Security for the full HTTPS vs HTTP comparison.

Confidential inference (TEE)

If your server has Intel TDX or AMD SEV-SNP hardware, you can offer confidential inference — prompts are encrypted to your enclave and even your host OS cannot read them.

1. Generate enclave keypair

arknet tee keygen

This creates ~/.arknet/keys/enclave.key — an X25519 keypair. Users encrypt prompts to this key.

2. Register TEE on-chain

arknet tee register --platform intel-tdx --quote-file /path/to/attestation.bin

This submits a RegisterTeeCapability transaction with your attestation quote and enclave public key.

3. Enable in config

[tee]
enabled = true
platform = "intel-tdx"
arknet start --role compute

Your node is now advertised as TEE-capable. Users sending prefer_tee: true requests are routed to you. TEE-verified jobs earn 1.5x emission rewards.

No TEE hardware? No problem. Standard compute nodes still protect prompts via Noise-encrypted P2P and protocol-level prompt isolation. TEE adds hardware-enforced confidentiality on top. See Privacy & Security for the full model.

Verify it's working

# Check node health
curl http://localhost:26657/health

# Check peers
curl http://localhost:26657/peers

# Check chain status
curl http://localhost:26657/v1/status

# List available models
curl http://localhost:26657/v1/models

Use the API

Once a compute node is serving models, any OpenAI client works:

from openai import OpenAI

client = OpenAI(base_url="http://localhost:26657/v1", api_key="local")
resp = client.chat.completions.create(
    model="meta-llama/Llama-3.1-8B-Instruct",
    messages=[{"role": "user", "content": "Hello!"}],
    stream=True,
)
for chunk in resp:
    print(chunk.choices[0].delta.content, end="")

Earning ARK

Bootstrap period (first 6 months): No stake required, no cost to users. Just start your node and earn ARK from block emission for serving free inference. This is how the token supply bootstraps from zero. See Tokenomics for details.

Once your node is serving verified inference:

  • Compute nodes earn 75% of the reward per job
  • Routers earn 5% for dispatching
  • Verifiers earn 7% for checking output
  • 3% is burned (deflationary)
  • 5% to treasury (governance-controlled)
  • 5% to delegators (pro-rata by staked amount)

Rewards are minted from the emission schedule. See Tokenomics for details.

CLI Reference

Every operator action is a CLI command. No raw transaction hex needed.

Wallet & transfers

# Create or show wallet
arknet wallet create
arknet wallet address
arknet wallet balance

# Send ARK
arknet wallet send --to 0x... --amount 1000000000

Staking

# Stake for a role (validator, router, compute, verifier)
arknet wallet stake --role compute --amount 50000000000000

# Begin unstaking (starts 14-day unbonding)
arknet wallet unstake --role compute --amount 50000000000000

# Finalize unbonding after 14 days
arknet wallet complete-unbond --role compute --unbond-id 1

# Move stake to a different node (1-day cooldown)
arknet wallet redelegate --role compute --to-node 0x... --amount 50000000000000

Gateway

# Register as a public gateway (HTTP)
arknet gateway register --url http://YOUR_IP:26657

# Register as an HTTPS gateway (1.2x reward multiplier)
arknet gateway register --url https://rpc.mynode.com --https

# Remove from gateway registry
arknet gateway unregister

Governance

# Submit a proposal (requires 10,000 ARK deposit)
arknet governance propose --title "Add Llama 4 Scout" --body "Proposal details..."

# Read body from a file
arknet governance propose --title "Add Llama 4 Scout" --body @proposal.md

# Vote on a proposal (yes, no, abstain, no-with-veto)
arknet governance vote --proposal 1 --choice yes

How governance works: Anyone with 10,000 ARK can submit a proposal. It enters a 7-day discussion phase, then a 7-day voting phase. Validators vote weighted by stake. Quorum = 33% of bonded stake. Threshold = 67% yes votes. If it passes, the action executes on-chain and the deposit is returned. If it fails, the deposit is burned.

Common proposals: add new models, adjust fee parameters, spend treasury funds. Architecture changes require a code release — governance only controls on-chain state.

TEE (confidential inference)

# Generate enclave keypair
arknet tee keygen

# Show enclave public key
arknet tee pubkey

# Register TEE capability on-chain
arknet tee register --platform intel-tdx --quote-file attestation.bin
